Google OAuth Configuration

Get the Client ID and Client Secret from Google Cloud Console: APIs & Services -> Credentials -> Create Credentials -> OAuth client ID -> Web application.

The Client Secret is accepted here, but AthletiSync stores it encrypted and never shows the saved value back in the UI. Leaving it blank keeps the current saved secret unchanged.

If you prefer, AthletiSync will still use the server environment variable GOOGLE_OAUTH_CLIENT_SECRET instead of the UI-saved secret when that env var is present.

Copy these exact values into Google Cloud when creating the OAuth client:

Google requires the origin to be just the base URL like https://example.org. The redirect URI is the full callback URL ending in /google/oauth/callback.

This does not need a public domain. It only needs to be reachable by the browser you use during Google sign-in. If you access AthletiSync at http://172.16.1.77, then these values can also use http://172.16.1.77.

Add Google Account Connection

Setup Instructions

Use this section to add the Google account AthletiSync should use for calendar access.

  1. Open https://console.cloud.google.com/.
  2. Create or select a Google Cloud project.
  3. Open APIs & Services -> Library.
  4. Search for Google Calendar API and click Enable.
  5. Open APIs & Services -> Credentials.
  6. Click Create Credentials -> OAuth client ID.
  7. If Google asks you to configure the consent screen first, do that and then come back to the credentials page.
  8. For application type, choose Web application.
  9. Under Authorized JavaScript origins, add {{ google_oauth_origin or "your server base URL" }}.
  10. Under Authorized redirect URIs, add {{ google_oauth_redirect_uri }}.
  11. After Google creates the OAuth client, copy the Client ID into Google OAuth Client ID on this page.
  12. Either paste the Google Client secret into the Google OAuth Client Secret field on this page, or set GOOGLE_OAUTH_CLIENT_SECRET on the server.
  13. Copy {{ google_oauth_redirect_uri }} into Google OAuth Redirect URI on this page if it is not already correct.
  14. Click Save Google OAuth Configuration.
  15. In the form below, enter a connection name such as District Calendar Manager.
  16. Click Sign In With Google.
  17. Sign in with the Google account that already has access to the calendar you want AthletiSync to manage.
  18. Approve the requested Google Calendar access.
  19. After Google sends you back to this page, AthletiSync saves that Google account as an auth profile.
  20. Repeat this section again if you want to connect a second Google account.
  21. Then use Add Calendar to save a separate calendar destination record.
  22. When you add the calendar destination, enter any label you want in Display Name, paste the exact Google Calendar ID, and select the connected auth profile.
  23. Use the Test Calendar Access button in the calendar list to confirm that the connected account has writer or owner access to that specific calendar.

This is where you add the Google account AthletiSync will use. It saves the Google account connection only. You will add the calendar destination separately in the next section.

Add Calendar

Calendar ID is not usually the visible calendar name. In Google Calendar, open the target calendar's Settings and sharing page, scroll to Integrate calendar, and copy the exact Calendar ID from there.

Examples: a primary calendar may use an address like yourname@gmail.com. A shared or secondary calendar often looks like abcd1234@group.calendar.google.com.

This creates the actual destination calendar record AthletiSync will sync into.
